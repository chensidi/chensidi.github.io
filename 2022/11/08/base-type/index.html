Symbol类型

Symbol是es6中新增的一种基本数据类型，也可以读成“符号”，其特点如下

  1. 不重复性，js中字符串，数字，和对象可以做到完全相等，但Symbol是不存在一模一样的两个值的，即便是 Symbol() == Symbol() 也是false，基于这个特点，当对象的属性为Symbol类型时，是不用担心某些属性会被覆盖的

  2. 不能使用 new 操作符，Symbol只能用做普通函数调用，当然可以用它的 call，apply，bind

  3. 可以全局注册Symbol，当采用普通函数调用时，Symbol得到的只是局部的符号值，下一次再调用时会重新注册，但Symbol提供了全局注册方法，Symbol.for，如果全局没有注册，则分配一个内存空间赋予首次的Symbol，下次再此注册时会直接使用内存中的值，所以在这种情况下，就能够得到两个完全相同的Symbol，这与第一点是有区别的，例如

    Symbol还提供了与for对应的一个方法，keyFor，这是用来获取传入for参数的方法，因为使用for方法，传入的参数最终都会转换为string类型，所以通过keyFor得到的数据，也都是string，当然如果调用for时没有传，那keyFor得到的也就是undefined

    keyFor只能接受Symbol类型，其余类型一概报错

  4. Symbol函数是接受参数的，该参数是作为该Symbol值的描述，为以后寻找该Symbol提供了追踪依据，参数一般传入的是字符串类型，但实际上能接受任何类型，只不过传入的类型最终都会变成字符串类型，例如数字，布尔会转成字符串，对象类型，会调用其toString方法，返回的结果作为Symbol入参的最终结果

  5. Symbol.toPrimitive，当一个对象存在该属性时，如果该对象作隐式类型转换，会优先调用该方法，它的存在使得优先级高于valueOf 和 toString，需要注意，这个属性值必须是函数

    也即是说，Symbol.toPrimitive对应的方法，接受的参数类型，是取决于作隐式转换时需要被转换成的数据类型，如果该对象没有这个属性，才去找有没有 valueOf 和 toString 方法

Number类型

Number 是js中很常见的类型之一，这里主要列举其容易混淆模糊的概念，数值转换，当Number构造函数传入一个值的时候，会进行数值转换得到一个Number类型的结果，以下是其转换特点总结

  1. 布尔类型，true -> 1, false -> 0
  2. 空字符串及全空格字符串，转换结果为0
  3. 字符串前后导空格会忽略，从第一个非空字符开始尝试转换
  4. 如果第一个非空字符串是 ‘+’, ‘-‘, 数字其中的一种，Number会尝试继续转换，否则直接返回NaN
  5. 如果开头以 ‘0x’ 开头，则尝试以16进制作为识别，转换为10进制数，如果’0x’后接的内容不符合16进制格式，则返回NaN
  6. ‘+’, ‘-‘ 不能结合 ‘0x’, 也就是说正负符号只对10进制有效，和16进制搭配结果反而是NaN
  7. 如果传入是对象，则按照隐式类型转换后得到的结果作为Number的入参
  8. 如果第一个有效字符为0（注意不是0x），则尝试以10进制转换，并忽略前面的所有0，前导0不能和0x搭配，否则返回NaN
  9. 其余情况均为NaN

es 中还有另外2种数组转换方法，parseInt, parseFloat，其中 parseInt 是更常用的将字符串转换为 Number 类型的方法，其特点总结如下

  1. 总是尝试将字符串转换为10进制整数
  2. 空字符及全空格字符串转换为NaN （Number中则是0）
  3. 前导空格忽略，从第一个非空字符开始（和Number很像），如果遇到非’+’,’-‘,数字开头，立即返回NaN
  4. 从第一个有效字符开始，一位一位往后走，直到遇到非数字字符或结束为止，也就是说截取字符串开头部分最长有效数字子串
  5. 忽略前导所有0（和Number一样）
  6. 正负号是可以搭配0x开头的16进制（与Number相反）
  7. 0开头不能与0x搭配（与Number相同）
  8. 0x开头会尝试识别为16进制

parseInt还可以接受第二个参数，也就是底数，说直接点就是明确告诉你，传入的字符串是什么进制的数，然后将该进制转换为10进制

parseFloat则是用来解析小数的，其特点汇总如下

  1. 只解析10进制
  2. 不能指定底数（可以传，但没用，不会报错）
  3. 从左到右，尝试解析浮点数据字符（数字和正负号和小数点），直到遇到非浮点类型字符为止或结束
  4. 如果第二次遇到小数点，也立即结束，返回前面的内容
  5. 正负号可以参与识别
  6. 如果筛选出来的浮点字符串末尾有0，则忽略全部结尾的0
  7. 其余与parseInt一致

String类型

字符串类型是js最常用的类型之一，其常用api有slice，substr，substring等，但这几个api却有比较独特的地方

  1. slice，这个方法接受两个参数startIndex，endIndex，其目的是从字符串中截取start到end中间的一段，但里面仍有值得研究的细节

  2. substr，该方法接受2个参数，startIndex，length, 目的是从开始索引位，向后取多少位

  3. substring，该方法接受2个参数，firstIndex, secondIndex，其目的是获取两个索引间的那一段字串，这个方法和slice有点类似，这也是很多初学者容易混淆的地方，这里的两个参数不一定非得谁比谁大，它们起的作用是得到一段左闭右开的区间即可

比较运算 “>” “<”

在js中，两个值的比较运算是很常见的，但很多人仅仅将其视为数字类型的大小比较，这只是其中的一方面，更多的比较原则归纳如下

  1. 两个值均为数字类型，则直接比较数学上的大小即可
  2. 两个值均为字符串类型，则按照字符串索引顺序，则尝试逐个比较其字符串对应码值大小
  1. 两个值其中一个是数字，另一个非数字，则非数字通过Number函数后转换为数字后，再做比较
  2. 两个值存在对象类型，则转为原始类型后比较
